# Technodruid.org — Phase 0 PRD (Foundation)

> **Status: COMPLETE as of 2026-05-18, iteration 27.** This PRD covered the foundation work — schema, layouts, components, doctrine fixes, and testing infrastructure — that everything later builds on. Phase 1+ work continues in sibling PRDs (see `sibling_prds:` frontmatter).
>
> **Split provenance:** this PRD was extracted from `PRD-20260516-technodruid-site.md` on 2026-05-18. Phase 0 ISCs + iter-1..27 LOG retained here verbatim; Phase 1/2/3/4 Key Files + ISCs moved to sibling per-phase PRDs.

## Goal

Build technodruid.org as described in `docs/information-architecture.md`: a credible, durable, self-hosted, doctrinally-aligned site for Technodruidism that distinguishes Canonical content (versioned, slow-changing, book-typography) from Living content (wiki-style, fast-changing, last-edited stamp), launches with a Phase 1 MVP of 6 pages, and grows through documented Phase 2 / Phase 3 expansions — all while adhering to the invariants in `docs/invariant_requirements.md`.

**Phase 0 scope (this PRD):** schema, layouts, components, doctrine fixes (font self-hosting, Comentario removal, print stylesheet, license files), Phase 0 doctrinal baseline (`pnpm verify` chain green), testing infrastructure (Playwright config + invariants/a11y/build specs). Closed as of iter-27 with all 41 ISC-FND-* milestones satisfied.

The existing CRT/terminal aesthetic (`src/styles/global.css`: scanlines, chromatic-aberration text-shadow, ASCII-art header in `src/components/Header.astro`) is preserved and adapted. Canonical pages get refined monospace typography (no new typeface introduced); Living pages get a wiki register on top of the same theme tokens.

## Phase Sequence

Iteration ordering. The work loop processes one subgoal at a time; subgoals within a phase may run in parallel as long as their files don't conflict.

**Phase 0 — Foundation.** Schema, layouts, components, doctrine fixes. Everything later phases build on. **Must complete before Phase 1 ISCs are picked up.**

**Phase 1 — MVP Launch.** Six pages per IA §8 launch list: `/`, `/codex`, `/practices/rite`, `/about/community`, `/about/origins`, `/about/contact`. Content placeholders allowed; site must build and pass invariants.

**Phase 2 — 60 days.** `/practices` landing, `/practices/wheel`, `/practices/glossary`, `/groves/charter`, `/library/reading`, `/contribute`. Wheel calendar adds ICS + RSS exports.

**Phase 3 — 6 months.** Groves directory, Library expansion (repair + links), Callings, About expansion (faq + section index), Archive, translation infrastructure, full accessibility audit.

**Phase 4 — Post-launch revisits.** Resolve `DECISION-PLAUSIBLE`, `DECISION-REPO` (GitHub → Forgejo evaluation), wire edit-page links if Forgejo lands, evaluate ActivityPub per IA §7.

## Testing Strategy

**Tiered:**

1. **Build verification** — `pnpm astro check` + `pnpm build` are run on every iteration's VERIFY. Both must exit 0.
2. **Invariant tests** — `tests/invariants.spec.ts` runs against `dist/` after build. Asserts the doctrinal invariants from `docs/invariant_requirements.md`. Re-runs every iteration.
3. **Smoke tests** — One spec file per phase (`tests/mvp.spec.ts`, `tests/phase2.spec.ts`, `tests/phase3.spec.ts`). Page exists → expected H1 → expected nav → expected footer → expected layout markers (TOC for canonical; edit-link for living).
4. **Accessibility tests** — `tests/a11y.spec.ts` (MVP) and `tests/a11y-full.spec.ts` (Phase 3). axe-core via Playwright. Zero serious/critical violations.
5. **Manual visual check** — Playwright trace + screenshot on every key page, retained in `test-results/` for diff review across iterations.

**TDD per iteration:** when an ISC produces code (not config or content), the test for it lands in the same iteration as the implementation file (Standard tier pair).
