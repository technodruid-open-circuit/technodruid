(2) **The "sticky TOC" bug was a `position:sticky` containing-block trap, not a sticky-element bug.** Removing `md:items-start` from the grid wasn't an obvious fix — the visible problem was "TOC scrolls away with content", but the fix is in the grid container, not the sticky element. (3) **Tailwind v4 cascade layers change the rules of CSS specificity reasoning.** The TOC gap-inconsistency root cause was `@layer utilities` (`:where(.space-y-1>:not(:last-child))` specificity 0,0,0) beating `@layer base` (`li{my-4}` specificity 0,0,1) for non-last items but not for `:last-child`. **In Tailwind v4, layer order beats per-selector specificity**: utilities layer wins over base layer regardless of which has the more specific selector. Mental model from v3 (where utility classes "win by being more specific") is wrong now. Diagnostic path: read computed styles via Playwright (`getComputedStyle(li).marginTop` per item), don't trust your own specificity arithmetic. (4) **The Astro-scoped `<style>` block is the right escape hatch when Tailwind utilities can't express the constraint.** `<ol class="space-y-1">` + global `li{my-4}` produced inconsistent margins; a Tailwind-only fix would need either a non-existent "make-this-element-not-receive-layer-base" utility, OR `last:m-0` (which kills bottom margin on XII but leaves the 0/4px asymmetry on I-XI vs the @base override for XII still bleeding).
